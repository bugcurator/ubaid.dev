Project Type Definitions (getProjectTypeClass())

This section outlines the purpose and classification of each project type used in the getProjectTypeClass() function. These tags help organize and display projects consistently across the UI.
1. client-based:- Projects built for clients based on their requirements, often under contract or freelance work.
2. personal:- Self-initiated projects for learning, experimentation, or portfolio building.
3. experience-based:- Projects completed during internships, jobs, or coursework — reflecting real-world experience.
4. sale:- Projects or templates specifically designed and ready to be sold or reused commercially.
5. maintenance:- Projects that are live and under ongoing maintenance, bug fixing, or feature updates.
6. archive:- Old or discontinued projects no longer actively maintained, kept for reference or history.
7. featured:- Highlighted projects that best represent your skills, creativity, or innovation.
8. ui-ux:- Projects focused heavily on user interface and user experience design, showcasing layout, flow, and visual design.
9. design-prototype:- High-fidelity mockups or prototypes created in design tools or with code to visualize ideas.
10. frontend:- Projects focused primarily on frontend development like HTML, CSS, JavaScript, and frameworks (React, Vue, etc.).
11. backend:- Projects built to handle server-side logic, databases, APIs, or authentication — using technologies like Node.js, Express, etc.
12. fullstack:- Complete projects covering both frontend and backend functionalities, often including databases and hosting.
13. devops:- Projects involving CI/CD pipelines, Docker, Kubernetes, cloud deployment, or infrastructure automation.
14. demo:- Quick mockups or proof-of-concept projects built to demonstrate a feature, idea, or skill.
15. team:- Projects completed as part of a team, highlighting collaboration, version control, and shared responsibilities.
16. solo:- Independently developed projects — showcasing individual design, development, and problem-solving capabilities.
17. clone:- Replications of real-world apps (e.g., Netflix, Amazon clone) for learning or practice purposes.
18. educational:- Projects built for teaching, tutorials, bootcamps, or learning purposes — including assignments and labs.
19. template:- Reusable layouts or components designed for quick development — ideal for landing pages or SaaS starters.
20. micro:- Small-scale utility apps, widgets, or components solving a single task or use case (e.g., to-do app, calculator).
